Task 3

Name Web Attacks
SQL Injection
Image Source
  • SQL Injection (SQLi) allows attackers to execute malicious SQL commands on a database, bypassing security measures.
  • It can result in unauthorized access, data theft, manipulation, or deletion, and potential compromise of the entire network.
  • SQLi is a highly dangerous vulnerability, affecting any website or application that uses SQL databases.
Source: https://www.acunetix.com/websitesecurity/sql-injection/
DDoS attack
Image Source
  • A DDoS attack disrupts a server, service, or network by overwhelming it with a flood of traffic from multiple compromised systems.
  • These attacks use botnets, networks of infected devices, to send massive amounts of traffic to a target, causing denial of service.
  • The attack traffic is difficult to distinguish from normal traffic because each bot is a legitimate Internet device.
Source: https://www.cloudflare.com/learning/ddos/what-is-a-ddos-attack/
Cross-Site Scripting
Image Source
  • Cross site scripting (XSS) injects malicious code into a vulnerable web application, targeting users rather than the application itself.
  • XSS attacks can compromise user accounts, activate Trojans, modify page content, and reveal session cookies, leading to impersonation and abuse.
  • There are two types of XSS: stored (persistent) XSS, which injects a script into the application, and reflected XSS, where the script is reflected off the application and activated via a link.
Source: https://www.imperva.com/learn/application-security/cross-site-scripting-xss-attacks/
Phishing
Image Source
  • Phishing is a scam where attackers impersonate trusted sources through communication like email, phone calls, or SMS to steal confidential information or make it inaccessible.
  • In a VPS environment, phishing often involves scam emails with links or attachments that lead to malicious software installation or requests for sensitive information, potentially compromising devices and accounts.
  • Once scammers gain access, they may steal information, money, or use compromised accounts to send further phishing attempts, often while posing as the victim.
Source: https://ovic.vic.gov.au/privacy/resources-for-organisations/phishing-attacks-and-how-to-protect-against-them/
Brute Force Attack
Image Source
  • A brute force attack involves repeatedly guessing usernames and passwords to gain unauthorized access to a system, often using automated tools.
  • Attackers may use bots and pre-obtained lists of credentials to systematically attempt access, making brute force attacks highly effective.
  • There are various methods of brute force attacks, including simple, hybrid, dictionary attacks, and more, each with different techniques for guessing or exploiting login information.
Source: https://www.imperva.com/learn/application-security/brute-force-attack/
Man-in-the-Middle
Image Source
  • A man-in-the-middle (MITM) attack occurs when an attacker secretly intercepts and potentially alters communication between a user and an application, often to steal sensitive information.
  • Common targets include users of financial applications, e-commerce sites, and other login-required services, with stolen information used for identity theft, unauthorized transactions, or further attacks.
  • A MITM attack is like a mailman opening and reading your bank statement before delivering it, making it seem like the communication was never intercepted.
Source: https://www.imperva.com/learn/application-security/man-in-the-middle-attack-mitm/
Session Hijacking
Image Source
  • Session Hijacking is an attack that exploits the web session control mechanism by compromising the session token, which is used to identify a user's connection to a web server.
  • Attackers steal or predict a valid session token to gain unauthorized access, bypassing normal authentication processes.
  • Common methods for compromising session tokens include predictable tokens, session sniffing, client-side attacks, and man-in-the-middle or man-in-the-browser attacks.
Source: https://owasp.org/www-community/attacks/Session_hijacking_attack
Remote Code Execution
Image Source
  • Remote Code Execution (RCE) is a cyber-attack where an attacker remotely gains control of a victim's system, allowing them to perform any action, such as deleting data or accessing sensitive information.
  • RCE attacks exploit vulnerabilities in software, operating systems, or web servers, enabling attackers to inject and execute malicious code on the compromised system.
  • The impact can be severe, such as a web server being controlled by an attacker, leading to defacement, data theft, or service disruption.
Source: https://www.aquasec.com/cloud-native-academy/cloud-attacks/remote-code-execution/
Cross-Site Request Forgery
Image Source
  • Cross-Site Request Forgery (CSRF) is an attack that forces authenticated users to submit unwanted requests to a web application where they are already authenticated.
  • CSRF exploits the trust a web application has in the user's authentication, tricking it into executing actions as the user without their consent.
  • This attack takes advantage of a web application's inability to distinguish between legitimate user requests and those initiated by an attacker.
Source: https://www.synopsys.com/glossary/what-is-csrf.html
Clickjacking
Image Source
  • Clickjacking is an attack where users are tricked into clicking on hidden, malicious elements instead of the intended interface, leading to unintended actions like downloading malware or providing sensitive information.
  • This attack is possible due to the use of HTML frames or iframes, which allow attackers to overlay a transparent, hidden interface over a legitimate webpage.
  • Variations of clickjacking, often referred to as UI redressing or UI redress attacks, exploit this technique to manipulate user interactions without their knowledge.
Source: https://www.kaspersky.com/resource-center/definitions/clickjacking
